Technologies Used

Frontend : React : The frontend is built with React which is a powerful framework for creating dynamic and interactive user interfaces. React's component-based architecture enables modular and reusable development, while its efficient rendering ensures seamless updates and real-time functionality, ideal for applications like Griddy.

Backend : FastApi : The backend is built with FastAPI, a high-performance framework for creating robust and scalable APIs. FastAPI's support for asynchronous operations ensures efficient handling of multiple requests, while its use of Python type hints enables automatic data validation and serialization. Each module in Griddy has its own dedicated API, ensuring independence between modules and seamless scalability for different functionalities.

Desktop : Electron : The desktop application is built with Electron.js, a framework that enables cross-platform development using web technologies. Electron.js provides a seamless integration of React for the frontend and FastAPI for backend communication, delivering a native-like experience on Windows, macOS, and Linux.

Key Features

Main Concept : Griddy is an innovative platform specializing in grid optimization problems, using advanced techniques to deliver efficient solutions. With a React frontend, FastAPI backend, Electron Desktop App, and the Gurobi solver, Griddy offers three unique modules for now for specific challenges. The problems presented below highlight Griddyâ€™s core capabilities, with more modules coming soon.

Grid Shortest Path : This module calculates the shortest path between two points on a grid, with the flexibility for users to add blocked squares. Griddy dynamically updates the path in real-time as blocked squares are added or removed. The grid size is also editable, allowing for custom configurations. If a user attempts to block the start or end grid cell, or if no path exists, Griddy provides a clear message to inform the user.This feature is perfect for applications that require real-time route adjustments, like navigation systems and network routing.

Grid Polynomial Regression : This module allows users to add points on a grid, representing the x/y axis, and Griddy will then attempt to fit those points using a polynomial. The system provides an equation for the polynomial and a preview plot to visualize the fit. To ensure accuracy, users must provide at least 5 points that are not too far apart, and no two points can overlap. These constraints help guarantee the precision of the polynomial approximation, making it ideal for applications in data fitting and predictive modeling.

Grid Network Coverage : This module allows users to build walls on a customizable-sized grid, simulating physical barriers in a network area. The user can then specify the number of 2.4GHz or 5GHz routers to be placed on the grid. Griddy will strategically place the routers to maximize coverage across the grid. If a wall is placed on a router, Griddy will automatically re-calculate and find the best new placement to maintain optimal coverage. This module is ideal for applications in network planning and optimization, ensuring efficient coverage even in complex environments.
